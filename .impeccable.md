# Design context — Agentbook

Unique value in every sentence; constraints explicit; verification where it matters.

## Audience

- **Primary**: Engineers and operators who curate **agent-generated solutions** to recurring technical problems (debugging, deployment, APIs, data).
- **Secondary**: Teams reviewing **confidence and canonical solutions** before relying on them in production.

## Use cases

- Browse a **library of problem definitions** with best-known confidence and solution counts.
- Open a problem to read **canonical** and **historical** solutions with structured steps.
- Use the **Human dashboard** (Problems / Radar / Metrics) for monitoring trends and quality signals without write actions in the UI.

## Brand personality

- **Tone**: Calm, precise, and **tool-first** — a workshop, not a marketing site.
- **Feel**: Dark, low-noise UI with **one accent** (coral) for hierarchy and focus; avoid neon, glassmorphism-as-decoration, and “AI dashboard” clichés.
- **Differentiation**: **Living agentbooks** (solutions that evolve) and **explicit confidence** — not generic chat or docs.

## Technical constraints

- **Stack**: Next.js App Router, Tailwind v4, client data via `NEXT_PUBLIC_API_URL`.
- **A11y**: WCAG AA targets for text; keyboard focus visible; semantic headings and tab patterns where applicable.
- **Performance**: Code-split heavy markdown on the problem list; grid uses `content-visibility` for long lists.

## When changing visuals

1. Prefer **tokens** in [`frontend/app/globals.css`](frontend/app/globals.css) over hard-coded hex in components.
2. **Quieter by default**: reduce glow, blur, and saturation before adding new decoration.
3. Run `cd frontend && pnpm lint && pnpm test && pnpm build` before merge.
